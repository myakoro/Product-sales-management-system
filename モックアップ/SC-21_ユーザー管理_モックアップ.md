# SC-21 ユーザー管理モックアップ

## 画面概要

- 画面名: ユーザー管理
- 目的: マスター権限ユーザーが、ユーザーアカウント（ユーザー名・権限・パスワード初期化）を一元管理する。
- 対象: マスター権限のみ（スタッフにはメニューも画面も表示しない）

## レイアウト

- ヘッダー: 既存画面と同じ共通ヘッダー（システム名 / ナビ / ユーザー情報 / ログアウト）
- 本文:
  - ページタイトル: 「ユーザー管理」
  - セクション1: 「ユーザー新規作成」
    - ユーザー名（テキスト）
    - 権限（マスター / スタッフ）
    - 初期パスワード（テキスト）
    - 「ユーザーを追加」ボタン
  - セクション2: 「ユーザー一覧」
    - フィルタ: すべて / マスターのみ / スタッフのみ
    - 一覧テーブル
      - ユーザー名
      - 権限
      - 作成日時
      - 操作（権限切替 / パスワードリセット）

## 想定動作（モック段階）

- 権限切替:
  - 実装時には master/staff の切替を行い、最後のマスターがいなくならないように制御する想定。
- パスワードリセット:
  - 実装時には一時パスワードを発行し、画面に1回だけ表示する想定。

## 備考

- 本モックアップは、要件定義書・詳細仕様書で定義した項目がUI上で表現できているかを確認するためのもの。
- 実際のDB更新や認証まわりの制御は、動的実装側（Next.js + Prisma + NextAuth）で行う。
