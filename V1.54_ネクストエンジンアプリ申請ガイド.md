# ネクストエンジン アプリ申請ガイド (V1.54)

ネクストエンジン デベロッパーネットワークで「メインアプリ（受注連携）」を申請する際の設定項目と、審査で必要と思われる情報をまとめました。

## 1. 基本情報設定

| 項目 | 設定値（案） | 備考 |
| :--- | :--- | :--- |
| **アプリ名** | Rinori 売上管理システム | 自由に変更可能です |
| **Redirect URI** | `https://product-sales-management-system.onrender.com/api/auth/nextengine/callback` | 本番環境のURLです |
| **アプリアイコン** | (同梱の `rinori_ne_app_icon_v1_54.png` を使用) | 75x75ピクセル |

## 2. API権限設定 (スコープ)

申請時に以下の権限（エンドポイント）の利用許可を選択してください。

*   **受注参照**: 受注伝票および明細情報を取得するために必要です。
    *   `receiveorder_base` (受注伝票)
    *   `receiveorder_row` (受注明細)
*   **店舗参照**: マッピング設定画面で「店舗名」を表示するために必要です。
    *   `shop` (店舗マスタ)

## 3. 審査用「利用目的」の記述案

審査時に記入が必要となる「アプリの概要・利用目的」のテキスト案です。適宜調整してご利用ください。

> **利用目的の記述例:**
> 自社で運用している売上分析システム「Rinori」において、受注・売上データの集計を自動化するために利用します。
> これまではネクストエンジンから「受注明細一覧」をCSVで手動ダウンロードし、本システムにアップロードしていましたが、API連携によりこの作業を自動化し、人為的ミスを削減しつつ迅速な経営判断（販路別・商品別のPL分析）を可能にすることを目的としています。
> 本システムは社内限定で利用するものであり、取得したデータは売上集計および分析のみに使用し、個人情報は保存しません。

---

> [!TIP]
> **HTTPSでのテストについて**
> ネクストエンジンの仕様上、テスト環境（localhost）でも **HTTPS (`https://localhost:3000/...`)** が必須です。
> アプリ登録時に「リダイレクトURI」としてローカル用URLも追加しておくと、開発中の検証がスムーズになります。
