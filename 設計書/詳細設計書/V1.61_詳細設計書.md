### 3-1. SKU→親コード変換
```typescript
function convertToParentCode(sku: string): string {
  const trimmed = sku.trim();
  
  // Rule 1: RINO- 形式
  const rule1 = trimmed.match(/^(RINO-[A-Z0-9]+)/);
  if (rule1) return rule1[1];
  
  // Rule 2: ハイフンなし形式
  const rule2 = trimmed.match(/^(RINO[A-Z]+[0-9]{3,4})/);
  if (rule2) return rule2[1];
  
  // Rule 3: その他
  return trimmed;
}
```

```typescript
function calcExclTax(inclTax: number, taxRate: number): number {
```

```typescript
async function getTaxRateForYm(targetYm: string, prisma: PrismaClient): Promise<number> {
  const taxRate = await prisma.taxRate.findFirst({
    where: { startYm: { lte: targetYm } },
    orderBy: { startYm: 'desc' }
```

### 3-4. 期間予算の自動配分
```typescript
function distributeBudget(total: number, months: number): number[] {
  const base = Math.floor(total / months);
  const remainder = total % months;
  const result = Array(months).fill(base);
  result[months - 1] += remainder; // 端数は最終月に加算
  return result;
}
// 例: distributeBudget(100, 3) → [33, 33, 34]
```

### 3-5. CSVパース堅牢化
```typescript
async function parseCSV(file: File): Promise<string[][]> {
  // 1. バイナリ取得
  const buffer = await file.arrayBuffer();
  const uint8 = new Uint8Array(buffer);
  
  // 2. UTF-8でデコード試行
  let text = new TextDecoder('utf-8').decode(uint8);
  let parsed = Papa.parse(text, { header: true });
  
  // 3. ヘッダー検証（文字化け判定）
  const hasValidHeader = parsed.meta.fields?.some(f => 
    f.includes('商品') || f.includes('SKU') || f.includes('受注')
  );
  
  // 4. Shift-JISで再試行
  if (!hasValidHeader) {
    const decoder = new TextDecoder('shift-jis');
    text = decoder.decode(uint8);
    parsed = Papa.parse(text, { header: true });
  }
  
  // 5. 不正クォート補正（空フィールド保護）
  text = text.replace(/([^,\t\r\n])""/g, '$1"');
  
  return Papa.parse(text, { header: true }).data;
}
```

### 3-6. Amazonフォーマットパース
```typescript
function parseAmazonNumber(value: string): number {
  if (!value || value === '￥0' || value === '0') return 0;
  let cleaned = value.replace(/[￥¥]/g, '');
  cleaned = cleaned.replace(/,/g, '');
  const num = parseInt(cleaned, 10);
  return isNaN(num) ? 0 : num;
}
```

### 3-8. Amazon CSV処理フロー
```typescript
interface AmazonCsvRow {
  productCode: string;
  asin: string;
  title: string;
  quantity: number;
  salesAmountInclTax: number;
  salesAmountExclTax: number;
  costAmountExclTax: number;
  grossProfit: number;
}

async function processAmazonCsv(
  rows: Papa.ParseResult<Record<string, string>>,
  targetYm: string,
  prisma: PrismaClient
): Promise<{
  validRows: AmazonCsvRow[];
  unregisteredAsins: { asin: string; title: string }[];
}> {
  const validRows: AmazonCsvRow[] = [];
  const unregisteredAsins: { asin: string; title: string }[] = [];
  const taxRate = await getTaxRateForYm(targetYm, prisma);
  
  for (const row of rows.data) {
    const asin = row['（親）ASIN']?.trim();
    const title = row['タイトル']?.trim() || '';
    const quantity = parseAmazonNumber(row['注文された商品点数']);
    const salesAmountInclTax = parseAmazonNumber(row['注文商品の売上額']);
    
    if (!asin) continue;
    
    const product = await prisma.product.findFirst({
      where: { asin: asin },
      select: { productCode: true, costExclTax: true }
    });
    
    if (product) {
      const salesAmountExclTax = Math.round(salesAmountInclTax / (1 + taxRate));
      const costAmountExclTax = quantity * product.costExclTax;
      const grossProfit = salesAmountExclTax - costAmountExclTax;
      
      validRows.push({
        productCode: product.productCode,
        asin, title, quantity,
        salesAmountInclTax, salesAmountExclTax,
        costAmountExclTax, grossProfit
      });
    } else {
      if (!unregisteredAsins.find(u => u.asin === asin)) {
        unregisteredAsins.push({ asin, title });
      }
    }
  }
  
  return { validRows, unregisteredAsins };
}
```

### 3-10. 除外キーワードフィルタリング
- CSVパース直後、SKU正規化やマスタ参照の**前**にフィルタリングを実行。
- `ExclusionKeyword` テーブルの全レコードを取得し、各行の識別子（NE: 商品コード, Amazon: ASIN）と比較。
- 一致したレコードは処理対象から除外し、`excludedCount` として集計結果に含める。
- エラーとしては扱わず、正常なスキップとして処理する。

### 3-11. 未保存変更の検知（商品予算設定）

### 3-12. 商品コード（SKU）のリネーム処理
商品コードが変更された際、以下の順序でアトミックに（トランザクション内で）実行する。
1. **新規レコード作成**: 新しい `productCode` で商品マスタを仮作成。
2. **関連データの更新 (`updateMany`)**: 以下のテーブルの `productCode` を新コードに一括書き換え。
   - `SalesRecord`
   - `MonthlyBudget`
   - `PeriodBudget`
   - `PeriodBudgetHistory`
   - `NewProductCandidate`
3. **旧レコード削除**: 元の `productCode` の商品を物理削除。
4. **コミット**: すべての更新が成功した場合のみ反映。

## 4. Web API インターフェース

### 4-1. 売上CSV取込 `POST /api/import/sales`
**Request** (multipart/form-data):
- `file`: CSVファイル
- `targetYm`: 対象年月 (YYYY-MM)
- `importMode`: 'append' | 'overwrite'
- `salesChannelId`: 販路ID (必須)
- `dataSource`: 'NE' | 'Amazon'
- `comment`: 備考 (任意)

**挙動**:
1. `salesChannelId` の存在チェック
2. 除外キーワード判定（SKU正規化前に実行）
3. `dataSource` に応じてパースロジックを分岐（NE / Amazon）
4. `overwrite` モード時: 同月・同販路の既存データ・履歴を物理削除
5. 各行を `SalesRecord` として登録
6. `ImportHistory` レコード作成（`dataSource` を記録）

**Response（正常完了）**:
```json
{
  "success": true,
  "importedCount": 150,
  "skippedCount": 3,
  "excludedCount": 2,
  "historyId": 42
}
```

**Response（未登録ASIN検出時）**:
```json
{
  "success": false,
  "error": "unregistered_asins",
  "unregisteredAsins": [
    { "asin": "B07W4B3RGM", "title": "商品タイトル..." }

}
```

### 4-6. 販路マスタ `GET/POST/PATCH /api/sales-channels`
- `GET`: 一覧取得（`?activeOnly=true` で有効のみ）

- `GET`: 一覧取得（最新100件、ページング対応、`dataSource` 列を含む）
- `PATCH /[id]/channel`: 販路一括変更 `{ "salesChannelId": 2 }`
- `DELETE /[id]`: 取込単位削除（Cascade削除で紐づく売上も削除）

### 4-8. 商品マスタ `GET/PATCH /api/products`
- `GET /api/products`: 商品一覧取得。
  - `?categoryId={id}`: カテゴリー所属商品のみ取得
  - `?categoryId=null`: 未所属商品のみ取得
- **`PUT /api/products/[code]`**: 商品情報の更新。
  - **商品コード変更対応**: `productCode` が現在の `code` と異なる場合、関連データ（売上・予算・候補）を新コードに付け替えるトランザクション処理を実行（詳細は3-12参照）。
- **`DELETE /api/products/[code]`**: 商品の削除。
  - **売上実績チェック**: `SalesRecord` が1件でも存在する場合は削除不可（400エラー）。
  - **関連データ一括削除**: 売上実績がない場合、紐づく `MonthlyBudget`, `PeriodBudget`, `PeriodBudgetHistory`, `NewProductCandidate` を一括削除して物理削除を実行。

### 4-9. 新商品候補 `GET/PATCH /api/products/candidates`
- `GET ?status=pending|ignored`: ステータス別候補取得
- `PATCH /[id]`: ステータス更新（`pending` ↔ `ignored`、一括管理中/管理外登録）

### 4-11. 商品別PL `GET /api/pl/products`
**Query Parameters**: `startYm`, `endYm`, `salesChannelId`

```json
    "categoryId": 1,
```

### 5-11. 商品マスタCSV取込の安全装置
- **カテゴリー照合ロジック**: 取込時の差分チェック（`/api/products/import/check`）において、CSVのカテゴリー名を現在の `ProductCategory` 項目と照合。DBに存在しないカテゴリー名が入力されている場合、その行を「エラー（警告）」とし、チェックボックスを無効化して登録をブロックする。カテゴリー名が空欄の場合は「未分類（NULL）」として扱う。
- **文字コード自動判別**: フロントエンド取込時に `TextDecoder` を使用。`Uint8Array` の先頭3バイトでBOM (0xEF, 0xBB, 0xBF) を判定し、なければ Shift-JIS と UTF-8 のデコード試行で判別する。

- **出力列**: コード, 名称, ASIN, 販売価格, 原価, 区分, ステータス, カテゴリー名
- **値の変換**: 区分: `own` → "自社", `purchase` → "仕入" / ステータス: `managed` → "管理中", `unmanaged` → "管理外"

### 6-8. 集計の論理結合ルール
- 同一項目内（ids）の複数選択は **OR条件**（SQLの `IN` 句）。
- 項目間（期間、販路、商品/カテゴリー）はすべて **AND条件**。
- `Budget` および `AdExpense` には `salesChannelId` が存在しないため、特定販路フィルタ適用時は以下の挙動とする:
  - **実績値**: 指定販路で集計。
  - **予算値・広告費**: 販路別データがないため、「全体(Total)」の値を表示、または達成率/営業利益を N/A とする。
  - 正確な達成率および営業利益は「販路: すべて」の時のみ算出可能。

### 6-9. 販路フィルタ適用時のUI制約
- 予算・達成率のチェックボックスを非活性化。
- 注釈: 「※販路別の予算データがないため、販路絞り込み中は予算比較を表示できません」。

### 6-10. 営業利益の表示制御
- 全体PL画面以外（商品別・カテゴリー別）では「営業利益」チェックボックスを非表示。

### 6-11. フロントエンドでのデータ整形（Recharts対応）
- APIから返却される「月別データの中の複数商品配列」を、Rechartsの `LineChart` / `BarChart` が処理可能なフラットなオブジェクト形式に変換する。
- 整形後イメージ: `{ periodYm: "2025-12", sales_RINO-FR01: 100000, sales_RINO-FR02: 80000, achievement_RINO-FR01: 110.5, ... }`
- `selectedIds` ステートに基づいて `<Line />` コンポーネントをループで動的生成。ラインのラベル（凡例名）は `商品名 + 項目名` とする。

### 6-12. 未分類の識別子
- APIの `ids` パラメータに `"unclassified"` を渡すと、`categoryId: null` の商品を集計対象に含める。

### 6-13. パフォーマンス
- 12ヶ月分のデータを1秒以内に描画。
- 大量データ（商品数100以上、期間24ヶ月以上）でも3秒以内に表示。

**配置**: `public/templates/sales_template.csv`
**エンコーディング**: Shift-JIS
```csv
商品コード,受注数,売上金額（税込）,商品名
RINO-FR010-X-BLK,10,50000,サンプル商品1
RINODO002BLK,5,25000,サンプル商品2
```

### seed.js（初期データ投入）
```javascript
const { PrismaClient } = require('@prisma/client');
const bcrypt = require('bcrypt');
const prisma = new PrismaClient();

async function main() {
  // 販路マスタ
  const channels = ['EC', 'モール', '卸', '催事ポップアップ', 'その他'];
  for (const name of channels) {
    await prisma.salesChannel.upsert({
      where: { name },
      update: { isActive: true },
      create: { name, isActive: true }
    });
  }
  
  // 広告カテゴリ
  const categories = ['Meta広告', 'Google広告', 'アフィリエイト', 'その他'];
  for (const categoryName of categories) {
    const existing = await prisma.adCategory.findFirst({ where: { categoryName } });
    if (!existing) {
      await prisma.adCategory.create({ data: { categoryName, isActive: true } });
    }
  }
  
  // 管理者ユーザー
  const adminPassword = await bcrypt.hash('admin123', 10);
  await prisma.user.upsert({
    where: { username: 'admin' },
    update: {},
    create: { username: 'admin', password: adminPassword, role: 'master' }
  });
}
```

## 9. バリデーション・エラー処理

### 売上CSV取込
| エラー条件 | エラーメッセージ |
|---|---|
| 販路未選択 | 「販路を選択してください」（取込ボタンdisabled） |
| CSVフォーマット不正 | パースエラー表示、取込中断 |
| 必須列不足（NE） | 「商品コード列が見つかりません」 |
| 必須列不足（Amazon） | 「（親）ASIN列が見つかりません」 |
| 全行未登録ASIN | 「取り込める商品がありません」 |
| 数値パースエラー | 該当行をスキップ、警告表示 |

### 商品マスタ（ASIN登録）
| エラー条件 | エラーメッセージ |
|---|---|
| ASIN重複 | 「このASINは既に他の商品に登録されています」 |

### 広告費
- 日付未入力: バリデーションエラー
- 金額が0以下: バリデーションエラー
- カテゴリ未選択: バリデーションエラー

### 広告予算
- 予算額に負の値を許可しない。
- 重複登録の防止（`@@unique` によるDBガード）。

### カテゴリー
- カテゴリー名: 必須、1〜50文字、重複不可。
- 削除時: 確認ダイアログで所属商品数を表示。

### 取込履歴
- 存在しない販路ID: 400 or 404エラー。

## 10. データ整合性

### Cascade削除
- `ImportHistory` 削除時: 紐づく `SalesRecord` を自動削除（Prisma `onDelete: Cascade`）。

### 上書きモードの削除範囲
- 「対象年月」かつ「選択された販路」のみを削除。他の販路・他の月のデータは破壊しない。
- 置き換え対象となった過去の取込履歴も物理削除。

### NE同期の二重取込防止
- `SalesRecord.externalOrderId` のユニーク制約で重複を防止。
- 同一期間・販路の再同期時は既存データを上書き（削除→再登録）。

### フィルタ適用時の保存整合性
- 検索やフィルタで表示を絞り込んでいる最中に「保存」を実行しても、非表示の商品の予算データが消失しないことを保証。

## 11. セキュリティ・制約

### NE連携
- **個人情報の保護**: 受注明細ベースでの取得を徹底し、個人を特定可能な情報は取得段階で除外。
- **トークンの自動更新**: 同期処理前にアクセストークンのリフレッシュを実行。
- **レートリミット対応**: API負荷制限（429エラー等）を考慮し、リトライや待機処理を実装。
- **通信中断**: ネットワークエラー時は同期を中断し、ユーザーにエラー内容を通知。
- **自動同期の除外**: 自動同期（Cron等）やキャンセルの自動追跡は、本フェーズでは除外・将来課題とする。

### SQLインジェクション対策
- Prisma ORMを使用し、パラメータ化クエリで実装。
