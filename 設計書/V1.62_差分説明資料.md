## 1. 変更内容

### 1-1. ヘッダー固定機能
**目的**: スクロール時も列名を常に表示し、データの見やすさを向上

**実装方法**:
- テーブルヘッダー (`<thead>`) に `position: sticky; top: 0` を適用
- 背景色を設定してデータ行と重ならないようにする
- z-indexを適切に設定して他の要素より前面に表示

**CSS例**:
```css
thead {
  position: sticky;
  top: 0;
  background-color: #00214d;
  z-index: 10;
}
```

### 1-2. 売上不足額列の追加
**列名**: 売上不足額  
**配置**: 売上達成率の右側  
**計算式**: `予算売上 - 実績売上`

**表示ルール**:
- 実績売上 ≥ 予算売上 の場合: 「達成済み」と表示
- 実績売上 < 予算売上 の場合: 不足額を表示（例: ¥50,000）

**TypeScript型定義**:
```typescript
interface BudgetVsActualRow {
  // ... 既存フィールド
  shortageAmount: number | null;  // null の場合は「達成済み」
}
```

**フロントエンド表示ロジック**:
```typescript
const formatShortageAmount = (row: BudgetVsActualRow) => {
  if (row.actualSales >= row.budgetSales) {
    return '達成済み';
  }
  const shortage = row.budgetSales - row.actualSales;
  return `¥${shortage.toLocaleString()}`;
};
```

### 1-3. 必要販売個数列の追加
**列名**: 必要販売個数  
**配置**: 売上不足額の右側  
**計算式**: `CEIL(売上不足額 ÷ 商品単価)`
- 商品単価 = `予算売上 ÷ 予算数量`（予算数量が0の場合は0）

**表示ルール**:
- 実績売上 ≥ 予算売上 の場合: 「達成済み」と表示
- 実績売上 < 予算売上 の場合: 必要個数を整数で表示（切り上げ）（例: 25個）

**TypeScript型定義**:
```typescript
interface BudgetVsActualRow {
  // ... 既存フィールド
  requiredQuantity: number | null;  // null の場合は「達成済み」
}
```

**バックエンド計算ロジック**:
```typescript
const calculateRequiredQuantity = (
  budgetSales: number,
  actualSales: number,
  budgetQuantity: number
): number | null => {
  // 達成済みの場合
  if (actualSales >= budgetSales) {
    return null;
  }
  
  // 予算数量が0の場合は計算不可
  if (budgetQuantity === 0) {
    return null;
  }
  
  // 商品単価を計算
  const unitPrice = budgetSales / budgetQuantity;
  
  // 売上不足額を計算
  const shortage = budgetSales - actualSales;
  
  // 必要販売個数を計算（切り上げ）
  return Math.ceil(shortage / unitPrice);
};
```

**フロントエンド表示ロジック**:
```typescript
const formatRequiredQuantity = (row: BudgetVsActualRow) => {
  if (row.actualSales >= row.budgetSales) {
    return '達成済み';
  }
  if (row.requiredQuantity === null) {
    return '-';
  }
  return `${row.requiredQuantity.toLocaleString()}個`;
};
```

## 2. API変更

### 2-1. GET `/api/budget/vs-actual`
**既存レスポンス**に以下のフィールドを追加:

```typescript
interface BudgetVsActualResponse {
  products: Array<{
    productCode: string;
    productName: string;
    budgetQuantity: number;
    actualQuantity: number;
    quantityAchievementRate: number;
    budgetSales: number;
    actualSales: number;
    salesAchievementRate: number;
    budgetGrossProfitRate: number;
    actualGrossProfitRate: number;
    // V1.62で追加
    shortageAmount: number | null;      // 売上不足額
    requiredQuantity: number | null;    // 必要販売個数
  }>;
}
```

**計算処理の追加**:
```typescript
// API Route内での計算例
const products = rawData.map(p => {
  const shortageAmount = p.actualSales >= p.budgetSales 
    ? null 
    : p.budgetSales - p.actualSales;
  
  const requiredQuantity = p.actualSales >= p.budgetSales || p.budgetQuantity === 0
    ? null
    : Math.ceil((p.budgetSales - p.actualSales) / (p.budgetSales / p.budgetQuantity));
  
  return {
    ...p,
    shortageAmount,
    requiredQuantity
  };
});
```
