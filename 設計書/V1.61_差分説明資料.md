# Rinori 売上管理システム V1.61 差分説明資料

本資料は、V1.6（統合版）から V1.61 へアップデートされた際の変更点をまとめたものです。

## 概要
V1.61では、運用上の利便性を向上させるため、以下の2つの主要な変更を行いました。
1. **商品コード（SKU）の変更機能**: マスタ編集画面から商品コードを書き換えることが可能になり、関連データも自動追随します。
2. **関連データの一括削除**: 売上実績がない場合に限り、予算などの紐づくデータを一括でクリーンアップします。

## 1. 要件定義の差分 (V1.6 → V1.61)

### 商品マスタ仕様 (要件定義書 7章)
- **[追加] 商品コードの変更可能化**: 
  - 従来は固定（PK）だった商品コードを編集可能に変更。
  - 変更時、以下の関連データが新しいコードに自動リネーム（付け替え）される：
    - 売上実績、月別予算、期間予算、期間予算履歴、新商品候補
- **[追加] 商品削除時の挙動**:
  - 売上実績がない商品に限り、紐づく予算データ等も含めて一括削除するよう拡張。

## 2. 詳細設計の差分 (V1.6 → V1.61)

### データベース整合性 (詳細設計書 3章)
- **[追加] 3-12. 商品コード（SKU）のリネーム処理**:
  - アトミックな（トランザクション内）リネーム手順を定義。
  - 新規作成 → 関連データ一括更新 (`updateMany`) → 旧レコード削除 のステップ。

### API定義 (詳細設計書 4章)
- **[変更] PUT /api/products/[code]**:
  - パラメータに `newProductCode` を受け取るよう拡張。
  - トランザクションによるデータ移行ロジックを実装。
- **[変更] DELETE /api/products/[code]**:
  - `MonthlyBudget` などの関連テーブルも `deleteMany` で一括削除するロジックを追記。
